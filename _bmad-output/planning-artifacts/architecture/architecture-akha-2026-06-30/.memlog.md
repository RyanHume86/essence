- (decision by user) AD Persistence Port: every surface/feature reads/writes user data ONLY through a per-domain store module; those modules are the ONLY code allowed to import the backend client (base44). New domain data gets its own store module — no feature reaches past the boundary to a backend OR to localStorage directly. BINDS: where every piece of state lives + who may call the backend. PREVENTS: surfaces independently picking different persistence (split-brain like today's tasks-in-Base44 vs win-count-in-localStorage) and a local-first migration touching every call site.

- (decision by user) AD Progress derived (Fork 4): today's progress is DERIVED from task state, not a separate counter.
