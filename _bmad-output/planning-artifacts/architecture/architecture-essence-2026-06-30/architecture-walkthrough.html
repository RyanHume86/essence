Design paradigm

Client-side SPA · ports & adapters at the data edge

Four layers, dependencies pointing one way — down. Each layer can be built independently because the layer below it is fixed.

Surfaces — pages + components
render & capture intent · never touch a backend directly
↓
Domain stores — useTasks · usePrefs
the ports · single funnel per domain · own cache, optimistic mutation, rollback
↓
Pure logic — src/lib
ordering · derivations · day-end · unit-tested, depends on nothing above
↓
Adapter — base44Client
the only module that imports the backend SDK · swappable

A separate transient reaction channel carries companion signals — orthogonal to persistence. State is derived; reactions are signaled.

What we inherited

Ratify the code; resolve only what was punted

The spine accepts the conventions the shipped code already proves, and spends its decisions on the forks the PRD/UX explicitly deferred to "the architecture phase."

Ratified from the code

  • React 18 / Vite 6 / Tailwind / shadcn / TanStack Query / framer-motion
  • All task I/O funnels through useTasks (optimistic + rollback)
  • Core logic in src/lib — Vitest is the only safety net there
  • JS/JSX only · @/ alias · tokens, no hardcoded hex

Debt the spine targets

  • Split-brain: tasks in Base44, win-count in localStorage
  • clearCompleted deletes — but the Archive must retain
  • RING_TARGET=8 — a denominator the wheel forbids
  • Schema drift: code writes priority/today/due_time not in the entity

Fork 1 — the biggest

How v1 holds the local-first door open

v1 stays on Base44 (PRD NFR2). But local-first is the documented target (NFR11). The real question: fix an invariant now, or bake in cloud assumptions?

A
Document local-first as future, no boundary — cheapest now, a rewrite later.
B
Persistence port — surfaces touch data only through per-domain stores; stores are the only code that imports the backend. Base44 is the swappable v1 adapter. Chosen.
C
Build local-first now — contradicts NFR2, over-builds for a solo v1.
Decision → AD-1

The port rule kills the split-brain by making "where does this live?" a store-module decision — and it's mostly formalizing the useTasks funnel that already exists.

Fork 3 — completed → archived

Stamp, never delete

Today's code deletes completed tasks; the PRD needs them retained as "what I did." One nullable field models the whole lifecycle.

Active— complete (set completed_at) → Completed-unswept— "I'm done for today" (stamp archived_at) → Archived
  • archived_at set = in the Archive (ordered by archived_at desc — the single sort key).
  • "I'm done for today" bulk-stamps; it never deletes. "Ended day" is an action, not a flag.
  • Archiving fires only on that explicit gesture — never by age or midnight.

Honest consequence: never-swept tasks linger dimmed across days — faithful to "manual-only," not a bug. → AD-7

Fork 4 — the second half of the split-brain

Derive progress; don't store it

A
Derive from tasks via a completed_at timestamp — today's progress = tasks completed today. Single source of truth, both deferred bugs vanish, doubles as nothing-stored. Chosen.
B
Keep the localStorage counter — no schema change, but stays a second source of truth with its midnight + double-count bugs.
Decision → AD-3

The winMoment counter is retired. The wheel is accumulation-only — no denominator, RING_TARGET=8 deleted, never role="progressbar". Trade-off: un-completing now shrinks the wheel (the honest behavior).

Beyond the big four

Recolor · reactions · shell

Creature recolor → AD-10

The art is a ~255KB, 470-fill painterly SVG. Recolor via a per-scheme SVG feColorMatrix (desaturate→tint) over one inlined art set — not per-scheme baked assets, not CSS hue-rotate.

Reaction channel → AD-9

A transient tier-tagged emitter (subtask/task/dayend). Reactions are signaled, never persisted — and the line is random within a tier, closing the variable-reward loophole.

Delivery shell → AD-1 / AD-12

An installable PWA → local-first future targets IndexedDB. The service worker is for install only and never registers push.

Invariants & rules — the durable heart

What a builder can't read off the code

AD-1 Persistence port
Surfaces read/write only through per-domain stores; stores are the only code that imports the backend. Swap = rewrite store internals, never callers.
AD-2 Single-funnel optimistic store
One query cache per domain; patch computed in both onMutate & mutationFn; rollback to ctx.previous; never network-gated.
AD-3 Derived read-state
Progress & carry-forward are computed from tasks, never stored. completed_at drives today-count; no-due-date is never carry-forward.
AD-4 Pure logic in src/lib
Ordering, derivations, day-end live as pure, Vitest-tested functions — never re-implemented inline in a surface.

Invariants & rules

Schema, lifecycle, exposure

AD-5 Task schema invariants
priority 1–5 · order · completed_at · archived_at; due_date date-only; drift reconciled; changes round-trip-verified.
AD-6 Personalization is local
Prefs + onboarded flag live in localStorage behind usePrefs; never on Base44.
AD-7 Archive: stamp, never delete
archived_at encodes the lifecycle; sweep bulk-stamps; Archive sorts by archived_at desc; archiving only on the explicit gesture.
AD-8 Focus exposure boundary
Focus shows exactly one task in effective order; full backlog only on Plan; manual order is a global override; comparator is total.

Invariants & rules

The restraint, made enforceable

AD-9 Reaction channel — signals, not state
Transient tier-tagged emitter; no-subtask task fires the task tier; line random within a tier; not revoked on rollback.
AD-10 Theming & recolor via tokens
7 schemes re-skin one primitive set; creature via feColorMatrix over one inlined art set; accent never body/link text.
AD-11 Reduced-motion & no-alarm
Every animation resolves to its end-state; info never silently dropped; completion needs check + strike, never colour alone; no red in the loop.
AD-12 Notifications off by construction
Service worker never registers push; opt-in toggle starts off; content bound by no-shame / no-absence.

  • Category UI — field kept, no UI; v1 is pure-priority.
